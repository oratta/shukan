# Smitch (Shukan) プロジェクトルール

## PR と issue の紐付け（必須）

- PR を作成するとき、対応する issue があれば PR 本文の先頭に **`Closes #<番号>`** を単体の行で書く（`Fixes` / `Resolves` も可）。マージ時に issue が自動クローズされる
- 複数 issue はカンマ区切り: `Closes #12, closes #34`
- **キーワードと `#番号` の間に語を挟まない**。「Closes 部分: issue #44」は無効で自動クローズされない（過去に #44 が取り残された実例）
- issue の一部だけに対応する PR は `Refs #<番号>` を使い、クローズさせない
- 対応 issue がない作業でも、ロードマップ上の Epic（#2〜#4, #10, #11 等）に関係するなら `Refs` で紐付ける
- PR マージ後は、自動クローズされたか確認し、`Refs` 運用の issue には進捗コメントを残す
