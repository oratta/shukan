### 6. 研究メタ情報フィールド

**調査結果：以下のフィールドは型に存在しません**

- `evidenceLevel` ❌ → 代わりに `confidenceLevel: 'high' | 'medium' | 'low'` が存在（行128）
- `sourceUrl` ❌ → 代わりに `article.sources[].url?` で個別文献の URL を保持
- `citation` ❌
- `studyDuration` ❌
- `participantCount` ❌
- `interventionProtocol` ❌

**存在する研究メタ情報：**

1. **`article.sources` 配列** — 各研究出典を記録：
   ```typescript
   sources: {
     id: number;      // 出典番号
     text: string;    // 引用テキスト（著者、年、論文タイトル、掲載誌）
     url?: string;    // DOI リンクなど
   }[]
   ```

2. **`confidenceLevel`** — 記事全体の信頼度：`'high' | 'medium' | 'low'`

3. **`calculationLogic`（optional）** — 計算根拠を構造化：
   ```typescript
   calculationLogic?: {
     health: CalcStep[];      // 健康効果の計算ステップ
     cost: CalcStep[];        // コスト削減の計算ステップ
     income: CalcStep[];      // 収入増加の計算ステップ
     positiveMood?: CalcStep[]; // 気分向上の計算ステップ
   }
   ```
   各 `CalcStep` は：
   ```typescript
   {
     label: string;    // ステップの説明
     value?: string;   // 値
     formula?: string; // 計算式
     result?: string;  // 結果
   }
   ```

**重要な特性：**
- 研究の実施頻度・プロトコル情報は**自然言語テキスト（researchBody）内に記述されており、構造化されていない**
- 例：「1日30分の中強度運動」「週6回以上自炊」などは researchBody 内の文字列として記述
- サンプル記事（禁酒）では、コメント行に「Heavy drinkers (&gt;350g/week)」のような詳細情報があるが、これはコード内コメントのみで、データ構造に組み込まれていない

## UXの質問に正直に答えます

## ① KPIは「人生でどのくらい得するか」が正しい

