# Smitch LP 指示書: 欲求起点オンボーディング訴求

## この文書の目的

Smitch LP を作成・修正する際に、プロダクトの最重要体験である「なりたい自分 -> 科学が習慣を導く」を、画面レベルで伝えるための指示書。

LP とオンボーディングでは、**Option 2: 欲求・感情トリガー起点**を主役にする。Option 1 のロールモデル起点は、世界観の補助としてのみ扱う。

## 決定事項

### 主方針

ユーザーに最初に見せる問いは、抽象的な人生モデルではなく、今の生活で発生している身近な欲求にする。

```text
今、いちばん変えたいことは？
```

理由:

- ユーザーが最初に持っているのは「ナポレオンのような人生」ではなく、「夕方になると動けない」「お金で諦めたくない」「上司や職場に振り回されたくない」といった具体的な不満
- 具体的な不満は LP 上で「これは自分のことだ」と認識されやすい
- 欲求カードから入ると、オンボーディング画面としてそのまま成立する
- Smitch の価値である「欲求を科学的に分解し、習慣へ翻訳する」が直感的に伝わる

### 表現の核

```text
入口は、身近な欲求。
出口は、科学的に選ばれた習慣。
```

または LP コピーとして:

```text
「変わりたい」を、きれいごとで終わらせない。
体力、お金、集中、自由、自信。
Smitch はあなたの欲求を科学的に分解し、
最初に選ぶべき習慣へ翻訳します。
```

## LP で伝えるべきプロダクト体験

LP では、Smitch が単なる習慣管理アプリではなく、以下の翻訳を行うアプリだと見せる。

```text
欲求・不満
  -> KPI / KPA への翻訳
  -> 習慣ランキング
  -> 習慣ごとのインパクト表示
```

例:

```text
夕方まで動ける体が欲しい
  -> 健康寿命 / 睡眠の質 / VO2max / 不調日数
  -> 就寝時刻を30分だけ固定する / 朝10分外に出る / 週2回の軽い筋トレ
  -> 集中時間 +2.1h、健康寿命 +2.4年、生涯コスト -82万円
```

## LP セクションへの反映

### Hero

Hero は従来の Smitch コアコピーを保つ。ただし右側の UI モックや補助カードには、欲求起点の選択画面を見せる。

推奨 Hero UI モック:

```text
今、いちばん変えたいことは？

[ 夕方まで動ける体が欲しい ]
[ お金で諦めたくない ]
[ 自分の時間を取り戻したい ]
[ 落ち着いて判断したい ]
```

Hero で避けること:

- 「どんな偉人のようになりたいですか？」を主問いにしない
- ナポレオン、エディソン、ナイチンゲールなどの実名を Hero の主役にしない
- 成功者崇拝、自己啓発セミナー風、野心の煽りに寄せない

### Problem-Solution

ユーザーの既存ペインは「続けられない」だけではない。より広く、日常の中で湧く小さく強い不満として扱う。

扱うべきペイン:

- 夕方になると何もできない
- 休日を寝て終わらせてしまう
- お金の不安で選択肢が狭まる
- 上司や職場に振り回される
- 家族に疲れた状態で接してしまう
- 頭がぼんやりして判断が鈍る
- 自分の時間がない

Solution の言い方:

```text
Smitch は、その不満を「気合い」の問題にしません。
あなたの欲求を、科学的に扱える指標へ分解し、
今のあなたに効きやすい習慣から提案します。
```

### How it works

How it works は 3 ステップで表現する。

1. `今、変えたいことを選ぶ`
2. `Smitch が科学的に分解する`
3. `効きそうな順に習慣を選ぶ`

既存の「なりたい自分を選ぶ / 科学が習慣を導く / 人生が動く」という構造は保つが、最初の画面は人生モデルカードではなく欲求カードにする。

画面例:

```text
01
今、変えたいことを選ぶ
夕方まで動ける体が欲しい
お金で諦めたくない
自分の時間を取り戻したい

02
Smitch が科学的に分解
睡眠の質 / 可処分時間 / ストレス耐性

03
効きそうな順に習慣を提案
就寝時刻を30分だけ固定する
朝10分外に出る
通知を2時間だけ切る
```

### Evidence

Evidence セクションでは、欲求が KPI に変換される例を具体的に見せる。

推奨デモ:

```text
選んだ欲求:
夕方まで動ける体が欲しい

Smitch が見る指標:
睡眠の質 / VO2max / 不調日数 / 集中時間

最初の習慣:
就寝時刻を30分だけ固定する

予測インパクト:
健康寿命 +2.4年
生涯コスト -82万円
集中時間 +2.1時間/日
```

ここで重要なのは、KPI をユーザーに選ばせるのではなく、Smitch が裏側で翻訳しているように見せること。

### CTA / Waitlist

Waitlist の「なりたい自分」自由入力は、欲求カードと同じトーンに寄せる。

推奨ラベル:

```text
今、いちばん変えたいことはありますか？
```

推奨 placeholder:

```text
例: 夕方まで動ける体が欲しい / お金で諦めたくない / 家族に良い状態で会いたい
```

## オンボーディング画面仕様

### Screen 1: 欲求選択

見出し:

```text
今、いちばん変えたいことは？
```

補足:

```text
Smitch は、あなたの「こうなりたい」を科学的に分解し、
最初に選ぶべき習慣を並べます。
```

カードは 8 枚を初期表示の上限にする。多すぎると選択負荷が上がる。

初期カード:

| 表示名 | 内部で見る主な指標 |
|---|---|
| 夕方まで動ける体が欲しい | 睡眠の質 / VO2max / 不調日数 |
| お金で諦めたくない | 生涯収入 / 貯蓄率 / 経済的レジリエンス |
| 自分の時間を取り戻したい | 可処分時間 / 通知制御 / 意思決定の質 |
| 落ち着いて判断したい | ストレス耐性 / 睡眠 / ジャーナリング |
| 家族に良い状態で会いたい | 家族時間 / 睡眠 / 感情制御 |
| 人に選ばれる自分でいたい | 自己効力感 / 清潔感習慣 / 対話時間 |
| すっきり考えられる頭が欲しい | 集中時間 / 睡眠 / 運動 |
| 理不尽に流されない自分になりたい | 経済バッファ / スキル習得 / ストレス耐性 |

### Screen 2: 欲求の深掘り

見出し:

```text
それが変わると、何が一番うれしいですか？
```

選択肢:

- 体が軽くなる
- 仕事の成果が出る
- 人間関係がよくなる
- 将来の不安が減る
- 自分に自信が持てる
- 自分で選んでいる感覚が戻る

この画面は、同じ欲求カードを選んだユーザーでも習慣ランキングを変えるための補助入力として使う。

### Screen 3: 翻訳結果

見出し:

```text
あなたが求めているのは、
「自分で選べる余白」かもしれません。
```

表示項目:

- 主要 KPI / KPA
- 補助 KPI / KPA
- 最初の 1 週間で狙うこと
- 推奨される最初の習慣

例:

```text
主要指標:
可処分時間

補助指標:
経済的レジリエンス / ストレス耐性 / 意思決定の質

最初の 1 週間:
睡眠固定 + 通知制御 + 10 分ジャーナリング
```

### Screen 4: 習慣ランキング

見出し:

```text
この順番で始めるのが、今のあなたには近道です。
```

カード例:

1. `就寝時刻を30分だけ固定する`
   - 効く指標: 睡眠の質 / 集中時間 / ストレス耐性
2. `朝10分だけ外に出る`
   - 効く指標: 体内時計 / 気分 / 日中の覚醒
3. `通知を2時間だけ切る`
   - 効く指標: 深い作業時間 / 自分の時間
4. `1日3行だけ書く`
   - 効く指標: 意思決定の質 / 自己効力感

## 欲求カードのコピー方針

本音をそのまま出すのではなく、ユーザーが選んでも恥ずかしくない言葉に翻訳する。

| 本音 | LP / UI 表示 |
|---|---|
| もっとお金が欲しい | お金で諦めたくない |
| 風邪をひかない体になりたい | 体調に予定を奪われたくない |
| モテたい | 人に選ばれる自分でいたい |
| 権力が欲しい | 理不尽に流されない自分になりたい |
| エネルギッシュな体が欲しい | 夕方まで動ける体が欲しい |
| 人生をコントロールしたい | 自分で選んでいる感覚を取り戻したい |
| 家族にイライラしたくない | 家族に良い状態で会いたい |
| 頭が悪くなった気がする | すっきり考えられる頭が欲しい |

## Option 1 の扱い

ロールモデル起点は破棄しない。ただし、LP とオンボーディングの主入口にはしない。

使いどころ:

- LP の下層で「Smitch が扱える人生モデルの広がり」を見せる
- 欲求カード選択後の翻訳結果として、より美しい人生モデル名を出す
- ブランドストーリーや記事で補助的に使う

例:

```text
夕方まで動ける体が欲しい
  -> 最後まで動ける人生

お金で諦めたくない
  -> 静かに自由でいる人生

人に選ばれる自分でいたい
  -> 関係の中で自信を持てる人生
```

## やってはいけないこと

- Hero の主問いを `どんな偉人のようになりたいですか？` にしない
- KPI 名を最初の選択肢にしない
- 欲求を自己責任・根性論・努力不足として扱わない
- `モテたい` や `権力が欲しい` をそのまま露骨に出さない
- 連続日数・バッジ・ランキングで煽らない
- 成功者の写真や偉人の肖像をメインビジュアルにしない
- SNS で見せるための自己改善に寄せない

## デザイン指示

- UI モックは実プロダクトのオンボーディング画面に見える密度にする
- 欲求カードは 2 列または 4 枚程度の抜粋で見せ、LP 縮小時にも読めるサイズにする
- KPI 翻訳は、タグや小さな指標カードで見せる
- 習慣ランキングは、番号付きカードで「順番に意味がある」ことを伝える
- 色は既存 impact 軸を使う: health / cost / income
- ただし欲求カード自体を派手な多色 UI にしない。Smitch の静かなトーンを保つ
- 写真を使う場合は、成功者風ではなく、日常の中で静かに自分を立て直している人物にする

## LP 制作用貼り付けブリーフ

LP に Smitch のオンボーディング体験を載せる場合は、以下を必ず反映する。

```text
Smitch のLPでは、Option 2: 欲求・感情トリガー起点を主役にする。
最初の問いは「今、いちばん変えたいことは？」。

ユーザーには KPI や人生モデルを直接選ばせない。
ユーザーは「夕方まで動ける体が欲しい」「お金で諦めたくない」
「自分の時間を取り戻したい」「家族に良い状態で会いたい」
などの身近な欲求カードを選ぶ。

Smitch はその欲求を、睡眠の質、健康寿命、可処分時間、
ストレス耐性、生涯収入、経済的レジリエンスなどの KPI / KPA に翻訳し、
その人に効きそうな順に習慣を並べる。

LP 上では、欲求カード -> KPI 翻訳 -> 習慣ランキング -> インパクト表示
の4段階が一目で分かる UI モックを入れる。

ロールモデル起点は補助扱いにする。偉人や成功者を入口にせず、
必要なら選択後の「あなたが向かっている人生モデル」として小さく出す。

トーンは、静かで知的。欲求は本音に近く、表示名は品位を保つ。
根性論、成功者崇拝、連続日数の煽り、SNS映え自己改善には寄せない。
```
