## Context

昨日レビューシート（`yesterday-review-sheet.tsx`）は直前のロングランで実装された。現在の UX にはレビュー体験として3つの改善点がある:
- 未入力習慣のみ表示 → 全習慣を見渡せない
- タップトグル式ステータス → 目的のステータスに辿り着くまで複数タップ必要
- システム絵文字のムード → プロジェクトの Lucide アイコン統一方針に反する

## Goals / Non-Goals

**Goals:**
- レビューシートを「昨日の全習慣を振り返る画面」として機能させる
- ステータス選択を直感的にする（ワンタップで目的のステータスに到達）
- ムード表現を Lucide アイコンに統一する

**Non-Goals:**
- 過去のレビュー内容を振り返るビューの実装（別 Change）
- daily_reflections の読み出し UI
- DB スキーマの変更

## Decisions

### D1: 全習慣表示 — page.tsx でフィルタを変更

page.tsx の `reviewHabits` スナップショットを `yesterdayUnreviewed` から `habits.filter(h => !h.archived)` に変更する。

バナーの表示条件とカウントは「未レビュー数 > 0」のまま維持。シートを開いた時だけ全習慣が見える。

**代替案**: シート内で completions から全習慣を再構築 → page.tsx の変更で十分なので不要

### D2: ステータスセレクト — インラインボタン群

タップトグル（サイクル式）をやめ、各習慣の行に3つの小さなボタン（completed / skipped / failed）を横並びで表示する。

```
[✓] [−] [✗]   ← 3ボタン、選択中のみハイライト
```

- `completed`: 緑ハイライト + Check アイコン
- `skipped`: グレーハイライト + Minus アイコン
- `failed`: オレンジハイライト + X アイコン
- 未選択（none）: 全ボタンが非ハイライト（アウトライン）
- 選択済みのボタンを再タップすると none に戻す

**代替案**: ドロップダウンセレクト → モバイルでは3つ程度ならボタン群の方がタップ数が少なく快適

### D3: ムードアイコン — Lucide 顔アイコン + カラーグラデーション

| Mood | Lucide アイコン | 色 |
|------|-----------------|-----|
| 1 | `Frown` | `text-red-400` |
| 2 | `Meh` | `text-orange-400` |
| 3 | `Minus` (CircleMinus) | `text-gray-400` |
| 4 | `Smile` | `text-lime-500` |
| 5 | `Laugh` | `text-green-500` |

Lucide の顔アイコンは線画で差が小さいため、色で段階を補強する。選択中はボーダー + 背景色でハイライト。

**代替案**: 数字のみ → 感情が直感的に伝わらない

## Risks / Trade-offs

- [全習慣表示で行数増加] → シートは `max-h-[90vh] overflow-y-auto` なのでスクロールで対応済み
- [Lucide の顔アイコンが5段階表現に不十分] → 色のグラデーションで補強。Mood 3 に `CircleMinus` は表情感が弱いが、中立を表すには妥当
