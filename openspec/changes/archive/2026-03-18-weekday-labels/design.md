## Context

HabitCard には今日のステータスボタン（大）と過去4日分のドット（小）が横並びで表示される。現状、各ドットに曜日情報が表示されていないため、どのドットがどの曜日に対応するか分からない。

## Goals

- 過去ドットの上に曜日略称を表示し、ドットの意味を明確にする
- ロケール（ja/en）に応じて曜日表記を自動切り替えする
- 既存のドットコンポーネントを変更せず、親側でラップして実現する

## Non-Goals

- 今日の StatusIndicator への曜日ラベル付与
- DayStatusDot コンポーネント自体の改修
- i18n メッセージファイルへの曜日文字列追加（ブラウザ API に委譲）
- ツールチップや日付表示の追加

## Decisions

### D1: ラベルの位置

**選択**: ドットの上（上側）にラベルを配置する。
**理由**: ドットの下はステータスを示す色が見やすい領域で、視線の流れが「曜日 → ドット」の順になるほうが自然。上に置くことでスキャンしやすい。

### D2: 今日のラベル有無

**選択**: 今日の StatusIndicator にはラベルを付与しない。
**理由**: 今日のボタンは大きく視覚的に区別されており、「今日」であることが明確。ラベルを付けるとレイアウトが崩れるリスクがある。

### D3: 曜日文字列の生成方法

**選択**: `new Date(day.date + 'T00:00:00').toLocaleDateString(locale, { weekday: 'narrow' })` を使用する。
**理由**: ブラウザ標準 API のため追加ライブラリ不要。`narrow` オプションで `月`・`M` のような短縮形が得られ、スペースを取らない。`T00:00:00` を付加してタイムゾーンオフセットによる日付ズレを防ぐ。

### D4: ロケールの取得方法

**選択**: `useLocale()` (next-intl) を使用する。
**理由**: アプリ全体で統一されているロケール管理方法。habit-card.tsx はすでに next-intl を使用しているため追加コストがない。

### D5: ラップ要素のスタイル

**選択**: 各ドットを `<div className="flex flex-col items-center gap-0.5">` でラップし、ラベルに `text-[9px] text-muted-foreground leading-none` を適用する。
**理由**: ドットとラベルを縦に並べる最小構成。`text-[9px]` はドットの直径に対してバランスが取れる最小サイズ。`leading-none` で余分な行間を除去してコンパクトに保つ。

## Risks

- `narrow` オプションの出力がブラウザ・OS によって異なる可能性がある（例: 英語で `M` でなく `Mo` になるケース）→ 実機確認で対処。
- ラベルを追加すると HabitCard の高さが若干増加する → デザイン上許容範囲。
