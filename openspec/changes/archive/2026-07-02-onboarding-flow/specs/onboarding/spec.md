# Spec Delta: onboarding (onboarding-flow)

## ADDED Requirements

### Requirement: プロフィール未作成ユーザーのオンボーディング誘導

システムは、ログイン済みかつ user_profiles 未作成のユーザーが保護ページ（`(app)` グループ配下）にアクセスしたとき、`/onboarding` にリダイレクトしなければならない（SHALL）。判定は `src/app/(app)/layout.tsx`（サーバーコンポーネント）で行い、`src/middleware.ts` は変更しない。

#### Scenario: 未作成ユーザーがホームにアクセスするとオンボーディングが表示される

- **WHEN** user_profiles 未作成のユーザーがログイン済みの状態でホーム（`/`）を開く
- **THEN** `/onboarding` にリダイレクトされ、画面[1]「どんな自分に切り替えますか？」が表示される

#### Scenario: 未作成ユーザーが他の保護ページにアクセスしてもオンボーディングが表示される

- **WHEN** user_profiles 未作成のユーザーがログイン済みの状態で `/discover` や `/settings` を開く
- **THEN** `/onboarding` にリダイレクトされ、画面[1]が表示される

#### Scenario: 作成済みユーザーがオンボーディングを開くとホームに戻される

- **WHEN** user_profiles 作成済みのユーザーが `/onboarding` をURL直打ちで開く
- **THEN** ホーム（`/`）にリダイレクトされ、通常のホーム画面が表示される

#### Scenario: 未ログインユーザーがオンボーディングを開くとログイン画面に戻される

- **WHEN** 未ログインのユーザーが `/onboarding` をURL直打ちで開く
- **THEN** `/login` にリダイレクトされ、ログイン画面が表示される

### Requirement: ステッププログレスの表示

オンボーディングの各画面の上部に、全4ステップ中の現在位置を示すステッププログレスを表示しなければならない（SHALL）。

#### Scenario: 画面を進めるとステッププログレスが進む

- **WHEN** ユーザーが画面[1]でKPIカードを1つ選び「次へ」を押す
- **THEN** 画面[2]が表示され、上部のステッププログレスが 4ステップ中2番目 を現在位置として示す

### Requirement: 画面[1] KPI選択（4カード単一選択）

画面[1]は、タイトル「どんな自分に切り替えますか？」、サブタイトル「まずは1つに集中しましょう。あとからいつでも変えられます。」、KPIカード4枚、補足「どの数字も科学研究にもとづいて計算します」、ボタン「次へ」を表示しなければならない（SHALL）。カードは1枚のみ選択でき（単一選択）、未選択では次へ進めない。カードの文言は確定文言どおりとする:

| 見出し（なりたい自分） | KPI名 | 説明文 |
|---|---|---|
| 長く健康でいられる自分へ | 健康寿命 | 習慣の積み重ねが健康寿命を何**分**延ばすかを記録します |
| 前向きな気持ちで過ごせる自分へ | 前向きな気持ちの時間 | 前向きでいられる時間が何**分**増えるかを記録します |
| お金で諦めない自分へ | 出費削減 | 生涯の出費をいくら（**円**）減らせるかを記録します |
| 稼ぐ力のある自分へ | 稼ぐ能力 | 生涯で稼ぐ能力がいくら（**円**）上がるかを記録します |

#### Scenario: 4枚のKPIカードが確定文言で表示される

- **WHEN** ユーザーが画面[1]を開く
- **THEN** タイトル「どんな自分に切り替えますか？」とサブタイトル「まずは1つに集中しましょう。あとからいつでも変えられます。」が表示され、4枚のカードに見出し（「長く健康でいられる自分へ」「前向きな気持ちで過ごせる自分へ」「お金で諦めない自分へ」「稼ぐ力のある自分へ」）とKPI名・説明文が確定文言どおり表示され、カードの下に「どの数字も科学研究にもとづいて計算します」が表示される

#### Scenario: カードを1つ選択すると次へ進める

- **WHEN** ユーザーがKPIカードを1枚タップして「次へ」を押す
- **THEN** 選択したカードが選択状態として強調表示されたのち、画面[2]「あなたの数字で計算します」に進む

#### Scenario: 別のカードを選び直すと選択が切り替わる（複数選択不可）

- **WHEN** ユーザーが「お金で諦めない自分へ」を選択した状態で「長く健康でいられる自分へ」をタップする
- **THEN** 「長く健康でいられる自分へ」のみが選択状態になり、「お金で諦めない自分へ」の選択は解除される（選択状態のカードは常に1枚だけ）

#### Scenario: 未選択では次へ進めない

- **WHEN** ユーザーがカードを1枚も選択していない状態で「次へ」を押そうとする
- **THEN** 「次へ」は無効（disabled）で押せず、画面[1]に留まる

### Requirement: 画面[2] プロフィール入力

画面[2]は、タイトル「あなたの数字で計算します」、サブタイトル「選んだKPIをあなた用に計算するために使います。」、入力項目（年齢=必須／性別=必須／国=必須・デフォルト日本／年収=任意）、年収欄の注記「未入力の場合は、年齢・性別・国の平均年収を使って計算します」、ボタン「次へ」を表示しなければならない（SHALL）。必須項目がすべて入力されるまで次へ進めない。

#### Scenario: フォームが確定文言で表示され国のデフォルトは日本

- **WHEN** ユーザーが画面[2]を開く
- **THEN** タイトル「あなたの数字で計算します」とサブタイトル「選んだKPIをあなた用に計算するために使います。」が表示され、年齢・性別・国・年収の4項目が表示され、国には「日本」が初期選択されており、年収欄に注記「未入力の場合は、年齢・性別・国の平均年収を使って計算します」が表示される

#### Scenario: 年収未入力でも次へ進める

- **WHEN** ユーザーが年齢・性別・国のみ入力し、年収を空欄のまま「次へ」を押す
- **THEN** 画面[3]に進む

#### Scenario: 必須項目が未入力では次へ進めない

- **WHEN** ユーザーが年齢または性別を未入力のまま「次へ」を押そうとする
- **THEN** 「次へ」は無効（disabled）で押せず、画面[2]に留まる

#### Scenario: 年齢に不正な値を入れると次へ進めない

- **WHEN** ユーザーが年齢欄に空文字・負の値・現実的でない値（例: 200）を入力する
- **THEN** 「次へ」は無効のままか、入力欄に妥当な範囲のみ受け付ける制御がかかり、画面[2]に留まる

### Requirement: 画面[3] 習慣のおすすめ（選んだKPIのプリセットから1つ以上選択）

画面[3]は、タイトル「「（選んだKPIのなりたい自分コピー）」に効く習慣」、サブタイトル「続けやすいものを1つ以上選んでください。あとから追加・削除できます。」を表示し、選んだKPIに対応する習慣プリセットのみをカードで表示しなければならない（SHALL）。各カードには1回あたりの効果を表示する。1つ以上選択するとボタン「この習慣ではじめる」が有効になる（複数選択可）。

#### Scenario: 選んだKPIに対応するプリセットだけが表示される

- **WHEN** ユーザーが画面[1]で「お金で諦めない自分へ」を選んで画面[3]まで進む
- **THEN** タイトル「「お金で諦めない自分へ」に効く習慣」が表示され、出費削減（cost_saving）向けの習慣プリセットのカードのみが一覧表示され、各カードに1回あたりの効果（例: 1回ごとに 出費 −600円）が表示される

#### Scenario: 1つ以上選択するとボタンが有効になる

- **WHEN** ユーザーが習慣カードを1枚タップして選択する
- **THEN** 「この習慣ではじめる」ボタンが有効になり、押すと画面[4]に進む

#### Scenario: 複数の習慣を選択できる

- **WHEN** ユーザーが習慣カードを2枚以上タップして選択し「この習慣ではじめる」を押す
- **THEN** 画面[4]に進み、選択したすべての習慣がリストに表示される

#### Scenario: 未選択ではボタンが無効

- **WHEN** ユーザーが習慣カードを1枚も選択していない
- **THEN** 「この習慣ではじめる」は無効（disabled）で押せない

### Requirement: 画面[4] 完了と一括書き込み

画面[4]は、タイトル「準備ができました」、本文「今日の1回から、あなたの「（KPI名）」が積み上がっていきます。」、選んだKPI（現在値 0）、選んだ習慣のリスト、ボタン「はじめる」を表示しなければならない（SHALL）。「はじめる」押下で、user_profiles 1行（tracked_kpis=選んだKPIキー）の upsert と、選んだプリセット分の habits の insert、プリセットの articleIds 分の habit_evidences の insert を一括実行し、成功したらホームへ遷移する。途中画面（[1]〜[3]）ではDBに書き込まない。

#### Scenario: 選んだKPIと習慣が表示される

- **WHEN** ユーザーが画面[3]で習慣を選んで画面[4]に到達する
- **THEN** タイトル「準備ができました」、本文「今日の1回から、あなたの「出費削減」が積み上がっていきます。」（選んだKPI名が入る）、選んだKPIが現在値 0 で表示され、選んだ習慣のリストが表示される

#### Scenario: はじめる押下でデータが書き込まれホームへ遷移する

- **WHEN** ユーザーが画面[4]で「はじめる」を押す
- **THEN** ホーム（`/`）に遷移し、画面[3]で選んだ習慣が今日の習慣リストに表示される（このとき Supabase 上には user_profiles 1行＝入力したプロフィール＋tracked_kpis=選んだKPIキー、選んだプリセット分の habits、プリセットの articleIds 分の habit_evidences が作成されている）

#### Scenario: 書き込みに失敗したらエラーが表示され再試行できる

- **WHEN** ユーザーが画面[4]で「はじめる」を押し、ネットワーク断などで書き込みに失敗する
- **THEN** 画面[4]に留まり、エラーメッセージが表示され、「はじめる」を再度押して再試行できる

#### Scenario: 完了後はオンボーディングに戻れない

- **WHEN** オンボーディングを完了したユーザーが再度 `/onboarding` を開く
- **THEN** ホーム（`/`）にリダイレクトされる

### Requirement: 途中離脱は最初からやり直し

オンボーディングの途中状態は永続化しない（SHALL NOT persist）。途中で離脱したユーザーが次回アクセスしたとき、オンボーディングは画面[1]から始まる。

#### Scenario: 途中離脱して再アクセスすると画面1から始まる

- **WHEN** ユーザーが画面[3]まで進んだ状態でブラウザを閉じ（またはリロードし）、再度ホームにアクセスする
- **THEN** `/onboarding` にリダイレクトされ、画面[1]が初期状態（カード未選択）で表示される（途中の入力内容は復元されない）

### Requirement: ja / en 両ロケールでの表示

オンボーディング全画面の文言は next-intl の messages（`onboarding` 名前空間）で管理し、ja は `docs/context/onboarding-screens.md` の確定文言（一字一句変えない）、en は ja からの翻訳を表示しなければならない（SHALL）。

#### Scenario: ja ロケールで確定文言が表示される

- **WHEN** locale が ja のユーザーがオンボーディングの4画面を順に進む
- **THEN** 各画面のタイトル・サブタイトル・カードコピー・注記・ボタンラベルが確定文言（「どんな自分に切り替えますか？」「あなたの数字で計算します」「準備ができました」「次へ」「この習慣ではじめる」「はじめる」等）どおり表示される

#### Scenario: en ロケールで翻訳文言が表示される

- **WHEN** locale が en のユーザーがオンボーディングの4画面を順に進む
- **THEN** 各画面のタイトル・サブタイトル・カードコピー・注記・ボタンラベルがすべて英語で表示される（未翻訳キーやキー名の生表示がない）
