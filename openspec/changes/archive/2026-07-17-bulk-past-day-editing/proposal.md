# Proposal: bulk-past-day-editing

Refs: issue #107（refs #2, #104）

## Why

#104（PR #105）で過去日の週ドットは「タップ一発でアクションシート」になったが、1日ずつ小さいドットを狙ってタップ→シートで選択→閉じる、の往復が日数分必要な構造は変わっていない。数日ぶりに開いて「この間ずっとやってた」人にとって、まとめて振り返るユースケースの操作コストが高い。また、編集可能枠（今日+過去4日）とロケット対象（6日以上前）の間に「ちょうど5日前の失敗日はどちらでも救えない」off-by-one の隙間が存在する。

## What Changes

- **一括編集シートの新設（issue 案1）**: 折りたたみカードの週ドット領域全体（ドット+余白）を1つの大きなタップ領域にし、タップすると直近日の行（日付 + [守れた/やった] [失敗] [スキップ] ボタン）が並ぶボトムシートが開く。狙う操作なしで連続入力できる
  - 対象日: 今日を除く過去7日のうち、習慣の対象日（曜日指定はその曜日のみ）
  - 選択済みステータスボタンの再タップで未入力に戻す（トグル）
  - quit の失敗タップ直後は行内に我慢率（resist_rate）4択チップを表示（任意入力）
  - 行末の「…」からその日1日分の既存アクションシート（メモ・我慢率入れ直し等）を開ける
- **過去日ドットの表示専用化**: ドット個別タップ（1日分アクションシート起動）を廃止。ドットは表示専用となり、領域タップ=一括編集シート起動に一本化 — **BREAKING**（#104 案A の入口を置き換え）
- **編集可能枠の7日統一**: 過去日の編集可能枠を「今日+過去4日」から「今日+過去7日」に拡大。詳細モーダルカレンダーのタップ可能日も7日に統一
- **ロケット対象の境界変更**: ロケット救済の対象を「6日以上前の失敗日（diff > 5）」から「8日以上前の失敗日（diff > 7）」に変更し、編集可能枠との隙間（5日前の失敗日が救えない）を解消。境界値は定数1箇所（例: `EDITABLE_PAST_DAYS = 7`）に集約
- **展開ビューの表示調整**: 展開ビューの情報構成（Life Significance / KPI影響 / 連続日数 / 節約額）は現状維持とし、下部ボタン列に「過去日をまとめて入力」導線を追加（一括編集シートへの第2入口）

## Capabilities

### New Capabilities
- `bulk-past-day-editing`: 直近数日の過去日ステータスをまとめて入力する一括編集シート。対象日の決定ロジック、行内トグル・我慢率チップ・1日分シートへの導線、起動入口（ドット領域タップ / 展開ビューのボタン）を定義する

### Modified Capabilities
- `habit-input-actions`: 「過去日の週ドットはタップ一発でその日のアクションシートを開く（案A）」の requirement を「ドットは表示専用、ドット領域タップで一括編集シートを開く」に置き換え。過去日の編集可能枠を7日と定義し、ロケット対象を8日以上前の失敗日とする

## Impact

- **UI**: `src/components/habits/habit-card.tsx`（DayStatusDot の表示専用化・ドット領域のタップ領域化・展開ビューのボタン追加）、新規 `src/components/habits/bulk-edit-sheet.tsx`（仮名）、`src/components/habits/habit-detail-modal.tsx`（tappableDates / rocketEligibleDates の境界変更）
- **状態管理**: `src/components/dashboard/dashboard-client.tsx`（一括シートの open 状態と配線）。データ更新は既存 `setDayStatus` の日単位呼び出しを再利用し、bulk API は追加しない（7件程度の逐次 upsert で十分）
- **ロジック/定数**: `src/lib/habits.ts` に編集可能枠の定数と対象日算出関数を追加
- **DB**: 変更なし（編集枠はクライアント側ルールのみ。`habit_completions` スキーマ・RLS はそのまま）
- **テスト**: `src/__tests__/habit-input-actions.test.ts` の過去日ドット関連シナリオ更新、一括編集シートの対象日算出・トグルロジックの新規テスト
