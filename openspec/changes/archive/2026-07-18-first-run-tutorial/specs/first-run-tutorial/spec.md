# first-run-tutorial Delta Specification

## ADDED Requirements

### Requirement: 起動条件と一度きりの表示制御

システムは、オンボーディング完了の書き込み成功時に localStorage（キー `smitch-tutorial`）へ `pending` を記録し、その後ホーム到着時にチュートリアルを一度だけ自動起動しなければならない（MUST）。チュートリアルの完走またはスキップで値を `done` に更新し、以後は自動表示してはならない（MUST NOT）。URL クエリ `?tutorial=1` が付いている場合は、`done` であっても再実行できなければならない（MUST）。表示制御はクライアント（localStorage）で完結し、DB を変更してはならない（MUST NOT）。

#### Scenario: オンボーディング完了直後に一度だけ起動する
- **WHEN** オンボーディングの [6] スタートで書き込みが成功しホームへ遷移する
- **THEN** 画面が落ち着いた後に「ようこそ」カード（使い方ガイドの開始）が表示される
- **WHEN** チュートリアルを完走またはスキップした後にホームを再訪問する
- **THEN** チュートリアルは表示されない

#### Scenario: クエリで再実行できる
- **WHEN** `done` 状態で `/?tutorial=1` を開く
- **THEN** チュートリアルが最初から起動する

### Requirement: ステップ構成と進行条件

チュートリアルは「ようこそ」カードに続き、次の6ステップをこの順で提示しなければならない（MUST）: [1] タップで達成 → [2] 再タップで取り消し → [3] 長押しでアクションシート → [4] 発見タブへの遷移 → [5] エビデンス記事からの追加説明 → [6] ゼロから自作の説明。[1][2] は対象習慣の当日ステータスが実際に completed / none へ変化したこと、[3] はアクションシートが開いて閉じたこと、[4] は `/discover` への実遷移によって進行しなければならない（MUST）。[5][6] は「次へ/完了」ボタンで進行する（MUST）。最終ステップ完了後は完走カード（ホームへ戻る導線）を表示しなければならない（MUST）。

#### Scenario: 実操作でステップが進む
- **WHEN** [1] で強調された達成ボタンをタップし当日が completed になる
- **THEN** [2]（再タップで取り消し）へ進む
- **WHEN** もう一度タップし当日が none に戻る
- **THEN** [3]（長押し）へ進む
- **WHEN** 達成ボタンを長押ししてアクションシートを開き、閉じる
- **THEN** [4]（発見タブへ）へ進む
- **WHEN** 強調された「発見」タブをタップして `/discover` に遷移する
- **THEN** [5]（記事からの追加説明）へ進む

#### Scenario: 説明ステップはボタンで進む
- **WHEN** [5] で「次へ」をタップする
- **THEN** [6]（ゼロから自作の説明）へ進む
- **WHEN** [6] で「完了」をタップする
- **THEN** 完走カードが表示され、「ホームへ」でホームに戻れる

### Requirement: スポットライトと操作制御

アンカー付きステップでは、対象要素の位置に穴（スポットライト）を開けた暗幕を実UIの上に重ねなければならない（MUST）。実操作で進むステップ（[1]〜[4]）では穴の内側の操作を実UIへ通し、穴の外側の操作はブロックしなければならない（MUST）。説明のみのステップ（[5][6]）では穴の内側もブロックする（MUST）。対象要素が画面外にある場合はスクロールして可視化し、スクロールやレイアウト変化に穴が追従しなければならない（MUST）。アクションシート等のオーバーレイ（z-50）が開いている間はチュートリアル表示を退避し、閉じたら復帰しなければならない（MUST）。

#### Scenario: 穴の中だけ操作できる
- **WHEN** [1] で穴の外（他の習慣カードやヘッダー）をタップする
- **THEN** 実UIには何も起きない
- **WHEN** 穴の中の達成ボタンをタップする
- **THEN** 通常どおり達成が記録される

#### Scenario: シート表示中は退避する
- **WHEN** [3] の長押しでアクションシートが開く
- **THEN** チュートリアルの暗幕・ツールチップは非表示になり、シートを自由に操作できる
- **WHEN** シートを閉じる
- **THEN** チュートリアルが次のステップで復帰する

### Requirement: 離脱手段

すべてのステップで「スキップ」導線を常時表示しなければならない（MUST）。Escape キーでもスキップできなければならない（MUST）。ただしシート退避からの復帰直後（500ms 以内）の Escape は、シートを閉じたキー入力の伝播による誤発火として無視しなければならない（MUST）。スキップ時は即座に `done` を記録し全表示を終了する（MUST）。

#### Scenario: いつでも離脱できる
- **WHEN** 任意のステップで「スキップ」をタップする
- **THEN** チュートリアルが終了し、以後自動表示されない

### Requirement: アンカー不在時のフォールバック

ホーム上の実操作ステップ（[1]〜[3]）は、対象アンカー（習慣の達成ボタン）が一定時間（4秒）見つからない場合、[4]（発見タブへ）まで自動でスキップしなければならない（MUST）。チュートリアルがアンカー不在で無限に停滞してはならない（MUST NOT）。

#### Scenario: 習慣0件でも詰まらない
- **WHEN** 習慣が1件もない状態でチュートリアルが起動する
- **THEN** [1]〜[3] は自動的に飛ばされ、[4]（発見タブへ）から案内が始まる
