## Requirements
### Requirement: ドット領域タップで一括編集シートが開く

システムは、折りたたみカードの週ドット領域全体（ドット・ドット間の余白を含む1つのタップ領域）のタップで、その習慣の一括編集シート（ボトムシート）を開かなければならない（MUST）。展開ビュー下部の「過去日をまとめて入力」ボタンからも同じシートを開けなければならない（MUST）。個別ドットは表示専用であり、ドット単位のタップ・長押し操作を持ってはならない（MUST NOT）。

#### Scenario: ドット領域のタップでシートが開く
- **WHEN** 折りたたみカードの週ドット領域（ドットの上でも余白でも）をタップする
- **THEN** その習慣の一括編集シートが開く
- **THEN** タップした位置がどのドットの上かによって挙動は変わらない（1日分のアクションシートは開かない）

#### Scenario: 展開ビューからもシートを開ける
- **WHEN** カード展開ビュー下部の「過去日をまとめて入力」ボタンをタップする
- **THEN** 同じ一括編集シートが開く

#### Scenario: ドット領域タップはカード展開と競合しない
- **WHEN** 週ドット領域をタップする
- **THEN** カードの展開/折りたたみはトグルされない（イベントはシート起動のみに使われる）

### Requirement: 一括編集シートの対象日

一括編集シートは、今日を除く過去6日（編集可能枠。今日を含めて計7日間＝1週間となり、週次習慣の時間単位と揃う）のうち習慣の対象日のみを、新しい日付が上になる順で行として表示しなければならない（MUST）。曜日指定（weekday / custom）習慣では対象曜日の日だけを行にする（MUST）。毎日・週次習慣では過去6日すべてを行にする（MUST）。各行はその日の現在ステータス（未入力/達成/失敗/スキップ/ロケット、quit 失敗時は我慢率）を反映しなければならない（MUST）。

#### Scenario: 毎日習慣は過去6日が並ぶ
- **WHEN** 毎日習慣の一括編集シートを開く
- **THEN** 昨日から6日前までの6行が、日付（例: 7/15 (火)）付きで新しい順に表示される

#### Scenario: 曜日指定習慣は対象曜日のみ並ぶ
- **WHEN** 月・水・金指定の習慣の一括編集シートを開く
- **THEN** 過去6日のうち月・水・金に該当する日だけが行として表示される

#### Scenario: 行は現在ステータスを反映する
- **WHEN** 過去日に `failed`（resist_rate=75）が記録済みの行を表示する
- **THEN** その行の「失敗」ボタンが選択状態で表示され、我慢率 75% が確認できる

### Requirement: 行内ステータス入力と連続入力

各行は「達成（quit では守れた系の既存文言）」「失敗」「スキップ」のステータスボタンを持ち、タップでその日のステータスを `setDayStatus` 相当の日単位更新として即座に記録・楽観反映しなければならない（MUST）。選択済みのステータスボタンを再タップした場合は未入力（`none`、レコード削除）に戻さなければならない（MUST）。ステータス入力でシートは閉じてはならず（MUST NOT）、複数日の連続入力を1回のシート起動で完了できなければならない（MUST）。

#### Scenario: 行のボタンタップでステータスが記録される
- **WHEN** 未入力の行の「達成」をタップする
- **THEN** その日が `completed` として記録され、ボタンが選択状態になり、シートは開いたままである

#### Scenario: 再タップで未入力に戻る
- **WHEN** 「達成」が選択状態の行でもう一度「達成」をタップする
- **THEN** その日のレコードが削除され `none` に戻る

#### Scenario: 別ステータスへの上書き
- **WHEN** 「達成」が選択状態の行で「スキップ」をタップする
- **THEN** その日が `skipped` に上書きされる

#### Scenario: 連続入力が1回の起動で完了する
- **WHEN** シートを開き、4行それぞれの「達成」を順にタップして閉じる
- **THEN** 4日分の `completed` が記録されている（シート起動1回+日数分のタップで完了）
- **THEN** 閉じた後の週ドット・連続日数表示に反映されている

### Requirement: quit 失敗時の行内我慢率チップ

quit 習慣の行で「失敗」をタップした場合、システムは即座に `failed` を記録した上で、その行内に我慢率4択チップ（0% / 25% / 50% / 75%）を表示しなければならない（MUST）。チップ選択でその日の `resist_rate` を保存しチップ表示を畳む（MUST）。チップは任意入力であり、選択せずに他の行を操作した場合やシートを閉じた場合も `failed` の記録は維持され `resist_rate` は null のままでなければならない（MUST）。positive 習慣の「失敗」ではチップを表示してはならない（MUST NOT）。

#### Scenario: quit の失敗タップでチップが出る
- **WHEN** quit 習慣の行で「失敗」をタップする
- **THEN** その日が `failed` として即記録され、行内に我慢率4択チップが表示される

#### Scenario: チップ選択で我慢率が保存される
- **WHEN** 表示されたチップから 50% をタップする
- **THEN** その日の `resist_rate` が 50 で保存され、チップ表示が畳まれる

#### Scenario: チップを無視しても失敗記録は残る
- **WHEN** チップを選択せずに別の行を操作する、またはシートを閉じる
- **THEN** `failed` の記録は維持され、`resist_rate` は null のままである

### Requirement: 行末の「…」から1日分のアクションシートを開ける

各行の行末に「…」ボタンを配置し、タップでその日付を対象とした既存の1日分アクションシート（メモ・我慢率の入れ直し・取り消し等を含む）を開けなければならない（MUST）。

#### Scenario: 「…」でその日のアクションシートが開く
- **WHEN** 7/14 の行の「…」をタップする
- **THEN** 一括編集シートが閉じ、7/14 を対象とした既存のアクションシートが開く

#### Scenario: アクションシートでメモを保存できる
- **WHEN** 開いたアクションシートで「メモ」を選び保存する
- **THEN** 7/14 の completion の note が更新される
